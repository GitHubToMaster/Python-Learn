pwd	:	显示当前文件的路径
ls	:	显示当前路径下的文件以及文件夹		
	-l	:	以列表的形式显示当前路径下的文件以及文件夹
	-h	:	配合-l，可以用来将-l中显示的文件大小更加可读(带了单位)
	-a	:	显示隐藏文件以及文件夹（包括子目录）

输出重定向： >	允许将命令执行的结果存放到文件中，本应该显示在终端上的内容显示到文件中
> 一个符号将会重定向到指定文件中，如果文件已经存在，内容将会被覆盖
>> 两个符号表示将会重定向到指定文件中，如果文件已经存在，将会进行追加

more 命令：当我们查看的内容太长的时候，可以使用more命令进行显示，每次只会显示1页，按下空格键可以显示下一页

|	:	Linux中的管道符，也就是指一个命令的输出可以作为另一个命令的输入，
			例如	ls -lha | more
				就是将 ls -lha 显示在终端中的内容存放到管道中，然后使用more读取管道中的内容
				这里管道左边是用来塞东西，右边是用来读取东西
touch	：	用来创建文件
mkdir	:	用来创建文件夹
rmdir	:	用来删除文件夹，文件夹必须为空，并且当前路径不在该文件夹里面
rm	:	用来删除普通的文件
		-r	:	用来第归删除文件
		-f	:	强制删除文件
		-i	：	表示以交互的方式删除文件
		
创建链接：使用ln命令进行创建
	硬链接：当我们使用ln命令创建一个链接的时候，首先会检查有多少个链接指向该文件（包括原来的文件名），当我们删除一个链接的时候，系统都会给该文件进行链接的计数，如果链接的数目不为0，那么文件依然存在，且文件改变的时候，所有链接指向的文件也会跟着改变！但是，当链接数目为0的时候，文件将会被删除！
	软链接：使用ln -s 源文件 目标文件	也就是创建一个新的文件，使得新目标文件指向原文件，当我们删除原文件之后，打开目标文件将会报错，提示找不到文件！
	
grep	:	用来进行文本搜索，表示我们知道文件的所在目录，根据文件名进行文件内容的查找和匹配
			grep "print" ex1.py 
			grep -n "print" ex1.py 	-n	将会高亮行数
			grep -n "^22" ex1.py 	表示搜索以22开头的文本，并且高亮显示
			grep -n "22$" ex1.py	表示搜索以22结尾的文本，并且高亮显示
			grep -n "[Nn]" ex11.py 表示搜索包含N或者n的文本，并且高亮显示，也就是说中间可以用来写正则表达式进行匹配
			
find	:	使用find进行文件的查找，用来在特定的目录中搜索符合条件的文件，
			find ./ -name "start.sh"	表示在当前目录下查找文件名为start.sh的文件
				参数如下：	-name	根据名字查找
							-size	根据文件的大小进行查找 例如 =5m -size 5M	
						-size -5k -size +5M 表示我们要找的文件大小大于5k小于5M的文件
							-perm 	查找当前目录下文件权限为给定数值的文件或者文件名
					find ./ -perm 888 查找当前目录下权限为888的文件或者目录

cp		:	拷贝文件
			-r		当要拷贝的文件是一个目录的时候，我们使用-r		cp	test	/lianxi	-r
			-v		显示拷贝进度
			-i		以交互的方式复制
			-a		拷贝过去之后，保留原目录的链接，以及文件属性，简单而言保留原文件的属性
			-f		已经存在的目标文件而不提示
mv		:	可以用来移动文件或者目录，也可以用来给文件重命名
			-v		显示拷贝进度
			-i		以交互的方式复制
			-f		禁止交互式的操作，如果覆盖也不会给出提示！
tar		:	可以用来打包文件
			-z		以gzip的格式打包或者解gzip结尾的包
			-v		显示打包或者解包进度
			-f		指定我们要将打包或者解包之后的文件存放在哪里
			-x		指定我们要解包
			-c		指定我们要打包
			-j		以bz2的格式打包或者解bz2结尾的包
			
gzip	:	可以用来对.tar结尾的文件进行压缩形成.tar.gz文件，或者对.tar结尾的文件进行压缩形成.tar.gz文件
			-r		对.tar结尾的文件进行压缩形成.tar.gz文件
			-d		对.tar结尾的文件进行压缩形成.tar.gz文件
			-C		用来指定压缩到哪个目录，后面跟着目录
zip		：	通过zip命令压缩文件不需要指定文件扩展名，文件扩展名默认为zip
unzip	：	将.zip结尾的文件解压
			-d		使用-d参数，后面跟上一个目录的名字，将文件解压到该目录，如果目录不存在会自动创建
			unzip -d test myzip.zip
				例子1：
					[root@localhost test]# tar -cvf all.tar ex[1-9].py
					[root@localhost test]# gzip -r all.tar
					[root@localhost test]# gzip -d all.tar.gz 
					[root@localhost test]# tar -xvf all.tar 
				例子2：以tar.gz的形式进行压缩和解压文件
					[root@localhost test]# tar -zcvf all.tar.gz ex[1-9].py 
					相当于例子1的前两句话
					[root@localhost test]# tar -zxvf all.tar.gz 
					相当于例子1的后两句话
				例子3：以bz2的压缩格式进行压缩和解压文件
					[root@localhost test]# tar -jcvf all.tar.bz2 ex[1-9].py 
					[root@localhost test]# tar -jxvf all.tar.bz2
				例子4：将压缩文件指定到哪个目录中(使用-C)
					[root@localhost test]# tar -zcvf all.tar.gz ex[1-9].py -C ./test1/
which		:	使用which查看命令的位置
